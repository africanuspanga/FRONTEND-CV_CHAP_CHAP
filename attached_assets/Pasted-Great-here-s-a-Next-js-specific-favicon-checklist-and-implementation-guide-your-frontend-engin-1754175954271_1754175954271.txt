## ‚úÖ Favicon Setup Checklist for **Next.js**

### üìÅ 1. **Place the favicon file correctly**

* [ ] Add your favicon file (`favicon.ico` or `favicon.png`) to the `/public` folder.

  ```
  /public/favicon.ico
  ```

This ensures it‚Äôs served from:

```
https://cvchapchap.com/favicon.ico
```

---

### üß† 2. **Add favicon to `<Head>` in `_app.js`**

In `pages/_app.js` or if using a custom `_document.js`, include this inside the `<Head>` tag:

```jsx
// pages/_app.js
import Head from 'next/head';

function MyApp({ Component, pageProps }) {
  return (
    <>
      <Head>
        <link rel="icon" href="/favicon.ico" />
        {/* Optional: Add PNG or Apple icon support */}
        {/* <link rel="icon" type="image/png" href="/favicon.png" /> */}
        {/* <link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png" /> */}
      </Head>
      <Component {...pageProps} />
    </>
  );
}

export default MyApp;
```

---

### üö´ 3. **robots.txt ‚Äî Do NOT block the favicon**

In `public/robots.txt`, make sure there is **no disallow** for the favicon path:

```txt
User-agent: *
Allow: /favicon.ico
```

If you don‚Äôt have a `robots.txt` file, you can skip this step ‚Äî Google will assume access is allowed.

---

### üß™ 4. **Test in browser**

* [ ] Run the site: `npm run dev` or deploy
* [ ] Visit `http://localhost:3000` or production URL
* [ ] Confirm the favicon appears in the browser tab
* [ ] Also visit: `https://cvchapchap.com/favicon.ico` ‚Äî it should open/download the image

---

### üß† 5. **Ask Google to Reindex**

* [ ] Go to [Google Search Console](https://search.google.com/search-console)
* [ ] Use the **URL Inspection** tool
* [ ] Enter your homepage: `https://cvchapchap.com`
* [ ] Click **Request Indexing**

---

### üßº 6. **Clear cache if needed**

If the favicon doesn‚Äôt update:

* [ ] Clear browser cache
* [ ] Hard refresh (`Cmd + Shift + R` on Mac or `Ctrl + F5` on Windows)
* [ ] Invalidate CDN or Vercel cache (if deployed there)
